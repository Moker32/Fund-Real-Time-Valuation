# Web 前端设计方案

**创建日期**: 2026-02-09

## 1. 项目概述

### 1.1 目标

为基金实时估值应用开发独立的 Web 前端，脱离 Flet 桌面框架的限制，实现更灵活的部署和更好的用户体验。

### 1.2 技术栈

| 分类 | 选择 | 说明 |
|------|------|------|
| 构建工具 | Vite 5.x | 极速开发体验，热更新快 |
| 框架 | Vue 3.4+ | Composition API |
| 语言 | TypeScript 5.x | 类型安全 |
| 路由 | Vue Router 4 | SPA 路由 |
| 状态管理 | Pinia | 轻量级状态管理 |
| HTTP 客户端 | Axios | 请求拦截、错误处理 |
| UI 样式 | CSS Variables + SCSS | 极简金融风配色 |
| 日期处理 | dayjs | 轻量级日期库 |

## 2. 设计风格

### 2.1 极简金融风

- **配色方案**: 黑/白/灰为主，涨跌用红色（涨）/绿色（跌）
- **字体**: 无衬线字体，数字使用等宽字体（tabular-nums）
- **布局**: 卡片式设计，大留白，信息层级清晰
- **动效**: 微交互，简洁的过渡动画

### 2.3 设计亮点

1. **侧边导航** - 固定左侧，点击切换内容区（基金/商品/新闻）
2. **市场概览卡片** - 顶部横跨，展示核心汇总数据
3. **双列布局** - 基金列表左主右辅，或网格排列
4. **数据行设计** - 每行一个基金，简洁明了
5. **涨跌视觉** - 用 ▲ ▼ 符号 + 颜色区分，一目了然
6. **底部状态栏** - 显示刷新倒计时、连接状态

## 4. API 接口规划

需要与后端确认或新增以下接口：

| 接口 | 方法 | 用途 |
|------|------|------|
| `/api/funds` | GET | 获取基金列表 |
| `/api/funds/{code}` | GET | 获取基金详情 |
| `/api/funds/estimate` | GET | 获取基金估值 |
| `/api/commodities` | GET | 获取商品行情 |
| `/api/health` | GET | 健康检查 |

**注意**: 可能需要改造后端，添加 REST API 路由。

## 5. 核心组件

### 5.1 FundCard

单个基金卡片组件，展示：
- 基金名称/代码
- 净值
- 估值/涨跌幅
- 微型走势图（可选）

### 5.2 FundList

基金列表组件：
- 搜索/筛选功能
- 基金卡片网格
- 排序功能

### 5.3 CommodityCard

商品卡片组件：
- 商品名称/代码
- 当前价格
- 涨跌幅
- 走势图

### 5.4 MarketOverview

市场概览组件：
- 持仓总值
- 今日涨跌
- 基金数量
